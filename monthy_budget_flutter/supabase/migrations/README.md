# Supabase Migrations

## How to create a new migration

1. Make your schema changes in the Supabase dashboard or locally
2. Generate a migration file:
   ```bash
   supabase db diff --schema public -f <descriptive_name>
   ```
3. The file will be created as `supabase/migrations/<timestamp>_<name>.sql`
4. Commit the migration file with your feature PR
5. Apply to remote: `supabase db push`

## Naming convention
- Timestamps are auto-generated by the Supabase CLI
- Use descriptive snake_case names: `add_meal_plans_table`, `alter_expenses_add_category`

## Local development
```bash
supabase start        # Start local Supabase
supabase db reset     # Apply all migrations from scratch
supabase db push      # Push migrations to remote
```

## Migration ordering
Migrations run in alphabetical (timestamp) order. The baseline migration
(`20260317000000_baseline_schema.sql`) documents the full schema as of 2026-03-17.
All subsequent migrations should only contain incremental changes.

## Existing loose SQL files
The `.sql` files in the parent `supabase/` directory (`schema.sql`, `products.sql`,
`coach_memory_schema.sql`, `household_activity_events.sql`) are the original
dashboard-applied scripts kept for reference. The baseline migration consolidates
them into a single versioned file.
