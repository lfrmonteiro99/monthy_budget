# Release Management Setup

This repository uses:
- CalVer versioning in `pubspec.yaml`
- `git-cliff` for changelog generation
- GitHub Actions for CI builds and releases

## Versioning
- versionName: `YYYY.M.PATCH`
- versionCode: `(YY * 10000000) + (MM * 100000) + PATCH`
- tag: `vYYYY.M.PATCH`

Example:
- `2026.3.1` -> `versionCode 260300001`

## Release
Run:

```bash
bash scripts/release.sh [--dry-run] [--allow-dirty] [--bump patch|minor|major]
```

The script:
1. Validates clean tree (unless --allow-dirty) + default remote branch
2. Detects release type from commits since last tag
3. Updates `pubspec.yaml`
4. Generates `CHANGELOG.md` section
5. Commits + tags + pushes

## CI release notes
Workflow uses the `CHANGELOG.md` section matching the tag as release body.
If missing, fallback to GitHub autogenerated notes.
